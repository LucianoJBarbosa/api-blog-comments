## Purpose

The ADRs in this repository exist to:

- preserve the technical rationale behind the main decisions
- document rejected alternatives and accepted trade-offs
- reduce ambiguity in future evolution of the baseline

## Structure

Each ADR follows the same structure:

- context
- decision
- consequences

The records are intentionally short. The goal is to preserve technical traceability without increasing documentation bureaucracy.
